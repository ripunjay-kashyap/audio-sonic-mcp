# 🎵 Audio Stem Splitter — MCP Server

Converts any YouTube URL into separated audio stems + a full sonic signature JSON, powered by **yt-dlp**, **FFmpeg**, **Demucs (htdemucs)**, and **librosa**.

---

## Architecture

```
YouTube URL
     │
     ▼
┌─────────────────────────────────────────────────────────┐
│  Stage 1 — Ingestion         validate_source()          │
│  Stage 2 — Download          yt-dlp  (audio-only)       │
│  Stage 3 — Convert           FFmpeg → 44.1kHz WAV       │
│  Stage 4 — Split             Demucs htdemucs            │
│  Stage 5 — Analyze           librosa (BPM, key, etc.)   │
│  Stage 6 — Vectorize         CLAP or librosa fallback   │
└─────────────────────────────────────────────────────────┘
     │
     ▼
  JSON Payload (stems + sonic_signature + telemetry)
```

## Installation

```bash
# 1. Clone or copy the project
cd audio-stem-mcp

# 2. Create virtual environment
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# 3. Install Python dependencies
pip install -r requirements.txt

# 4. (Optional) Enable CLAP vibe vectors — requires ~4GB download
pip install transformers torch torchaudio
```

---

## Running the Server

```bash
# Direct (for testing)
python server.py

# With custom stems output directory
STEMS_ROOT=/data/stems python server.py
```

| Parameter | Type | Description |

## Model Comparison

| Model | Quality | Speed | Stems |
|-------|---------|-------|-------|
| `htdemucs` | ★★★★☆ | Fast | 4 |
| `htdemucs_ft` | ★★★★★ | Slow | 4 |
| `htdemucs_6s` | ★★★★☆ | Medium | 6 (+ guitar, piano) |
| `mdx_extra` | ★★★★☆ | Medium | 4 |

> **Tip**: Use `htdemucs` for speed, `htdemucs_ft` for maximum quality.

## CLAP Vibe Vector

The vibe vector is a 512-dimension semantic audio embedding generated by the [LAION CLAP model](https://huggingface.co/laion/larger_clap_music_and_speech). It maps audio into the same latent space as text, enabling:

- Semantic similarity search ("find tracks that sound like this")
- Genre/mood clustering
- Cross-modal retrieval (audio ↔ text ↔ image)

If CLAP is not installed, the server falls back to a librosa-derived mel+MFCC embedding.
